# Token Budget Rules

harness-stack 第一版默认节约 token。

## 原则

1. 公共前言极简
2. 只加载当前 skill 需要的信息
3. 小任务不走重流程
4. 输出结构固定，避免冗长散文

## 建议预算

- 单个 `SKILL.md` 目标：8-12KB
- 单个 `SKILL.md` 上限：25KB

## 上下文使用

- 优先读摘要，再读细节
- 优先读必要文件，不做全量加载
- 默认单会话完成 `harness-apply` 与 `harness-qa`

## 什么时候该拆

如果某个 skill 出现以下情况，应拆分模板或共享片段：

- 文档明显超过上限
- 出现与当前能力无关的大量说明
- 多个 skill 重复同一段大文本

## 第一版不做

- 遥测注入
- 自动升级提示
- 长期 session 检测
- 大量运行时初始化信息
