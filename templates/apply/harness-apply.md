## 示例（参考写法）

以下示例只演示写法，不限制真实内容。

### 示例场景

按 `harness-eng` 方案，为设置页增加“退出登录”按钮。

## 本轮目标

让已登录用户可以从设置页主动退出当前会话，并回到登录页。

## 实现范围

- 设置页新增退出登录按钮
- 复用或补齐最小登出逻辑
- 成功后清空本地会话并跳转 `/login`

## 已完成项

- 设置页已增加退出登录按钮
- 点击按钮后会触发登出逻辑
- 成功后会跳转到 `/login`
- 基础 lint 与 session 相关测试已通过

## 关键改动点

- `src/pages/settings.tsx`：增加退出登录按钮与点击事件
- `src/lib/session.ts`：补齐清理 token 和本地会话的方法
- `src/router/index.ts`：确认登出后跳转登录页的路由处理

## 偏差说明

- 原计划默认复用统一 `logout()` 方法，但实际代码中只有零散会话清理逻辑，因此本轮在 `session.ts` 中补了一个最小公共登出方法

## 未完成项

- 未补退出前确认弹窗
- 未覆盖移动端窄屏下的按钮布局检查

## 风险与待验证点

- 需要确认内存中的 user store 是否和 localStorage 一起清空
- 需要验证退出后刷新页面是否还会残留旧用户名

## 交给 harness-qa 的输入

- 重点验证：
  - 设置页按钮是否可见且可点击
  - 点击后是否跳转 `/login`
  - 退出后刷新页面是否残留旧登录态
  - 访问受保护页面是否会重新要求登录
- 已执行的基础命令：
  - `pnpm lint`
  - `pnpm test --filter session`
