# 产品定位

## 一句话

harness-stack 是一套面向中文开发者的轻量 AI 开发骨架：用 PRD/工程双重评审、可执行任务化开发、QA 验证回炉与检查点快照，形成可持续迭代的工作流。

## 解决的问题

常见 AI coding 工作流有两个问题：

1. 太散：没有明确的评审、执行、验证、收尾顺序
2. 太重：一上来就带很多角色、很多工具、很多状态文件

harness-stack 的目标不是复制这些系统，而是把真正高价值的部分收敛成一条短链路。

## 第一版聚焦

第一版主链路只聚焦 5 个核心命令：

- `harness-prd`
- `harness-eng`
- `harness-apply`
- `harness-qa`
- `harness-checkpoint`

此外补一个轻量续接入口：

- `harness-resume`

## 设计原则

### 中文优先

对外文案、输出模板、skill 正文全部中文。

### 轻量优先

不做大而全平台，先把主链路跑顺。

### 闭环优先

开发不是写完就结束，开发后必须经过 QA；如果 QA 暴露失败项或高风险未验证项，应先回到 apply 修复，再进入 checkpoint。

### 模板驱动

复用共享片段，降低 prompt 漂移。

### 节约 token

公共前言极简；按需加载；尽量避免长而重的说明。
